Header File Docs

To make module development as easy as possible, Modules can be written like a C header file, or at least as close as it could be

Types

Types are the least C-like, just because it needs a lot more information than a typical C declaration provides.

// Syntax is as follows:
//typedef [name], [letter id], [unconnected icon], [connected icon], [colour], [output], [input tag type];
typedef bool, b, circle-thin, circle, #FF0000, %, checkbox;

The names auto and exec are already taken, as well as the letters ids g and e

Name

The name of the type

Letter ID

The id that the editor uses to build the nodes, can be anything but I'd recommend using a letter

Unconnected Icon

The Font Awesome 4 Icon to use on unconnected nubs

Connected Icon

The Font Awesome 4 Icon to use on connected nubs

Colour

Colour that represents that type, it will influence the icon, gradients, and lines connecting nubs

Output

The format for values, where % will be replaced. This is for situations where the language uses symbols to distinguish the type, such as putting 0x before a number to indicate it's a hex number, or putting "" around letters to indicate it's a string.

Input Tag Type

Type attribute for the input tag, to prompt users for a constant.

You can create an enum type by using | to separate the options, like Yes|No|Maybe

Nodes

The easiest type of node to add is an in-language function, where you just declare the function like so

FILE* fopen(char* filename, char* mode);
void rewind(FILE* stream);

If the function is defined, everything inside the brackets will be the node's output

The parameter names can be put in to insert its code wherever needed.

// auto means to use a generic nub that accepts any type
bool Equal(auto a, auto b)
{
    (a == b)
}
// The code inside is trim()'d so the white-space around it won't affect things

Declarations can be put before the function definition to change the type of node

- programmable

// Programmable nodes will first run the code inside and output whatever's returned
// This can be used to check if certain nubs are connected and change the output accordingly
programmable bool ThatFunction(bool A)
{
    if (!A)
    {
        return "The A input is not connected";
    }

    return "The A input is connected";
}

Here's the documentation for the node language

- _type[n]

// _type can be used to manually define the type of the node
// In this case I want my Add node to be treated as a type 4/macro node, used for operators
_type4 auto Add(auto _a, auto _b)
{
    (a + b)
} // Add (+)
// The (+) in the name is very important, macro nodes will use the symbol inside the parenthesis as the node's name

Here's a link to an explanation of the different node types

Parameter modifiers can be applied to change the nub

void CalculateStringLength(string _input, const bool include_velocity, float probability_of_blue_screen=0.5);

_Name

Adding _ to the start of the variable name means that the name will be blank on the node, but can still be referenced without the _ when writing the node's output.

bool Equal(auto _a, auto _b)
{
    (a == b)
}

const

const means that the nub will be replaced with the input tag for that type

=value

adding = to the end of the parameter name creates an optional nub, and what comes after = is the default value.

Also, they can be combined, but some might not work with others

If a comment is put at the end of the function, it will replace the name. With this you can include any character in the name

bool BoolOr(bool a, bool b)
{
    (a || b)
} // Or (||)

int IntOr(int a, int b)
{
    (a | b)
} // Or (|)

Multiple named outputs can be added as well, though it's not very C

float out_float, bool out_bool ThatFunction(float in_float, bool in_bool);

If the function has the start keyword, then it will spawn when a new document is created.

start exec Begin()
{
	%c0
}

If you need to, the custom keyword can be used to insert a node in the module-file format

custom If, 1, if (%0)\n{\n%o0\n}\nelse\n{\n%o1}\n%c2, e.;b.Condition;o.;e.True;e.False;e.Then;, jsref_if.asp

See the module-file docs for more info on this format

Structs/Classes

'struct' and 'class' mean the same thing here, it creates a structure that can contain properties and methods.

// Properties will have Get and Set nodes automatically created
// But if you don't want a Set node created, use the 'const' keyword
struct Vector
{
    float x;
    float y;
    const float z;
    float GetLength();
};

Properties used by the editor don't have a type to distinguish them from actual properties

struct MyStruct
{
    float theFloat;
    bool theBool;

    colour = #FF0000;
    prop-fmt = %this->%name;
    func-fmt = %this->%name(%params);
};

prop-fmt

The format for accessing properties on this struct, where '%this' is replaced with the reference, and '%name' with the property name

func-fmt

The format for calling methods on this struct, where '%this' is replaced with the reference, '%name' with the method's name, and '%params' with the parameters

The struct itself can have the 'const' keyword, which means it won't create break and make nodes, just a Get node for the struct's name.

// This is useful for languages with built-in classes, like JavaScript and C#
const struct console
{
    void log(auto Value);
    void clear();
};

If the struct's name is Settings, it will be treated like the settings area in the module file format

struct Settings
{
	doc-link=https://link-to-documentation
	separator=;\n
	...
};
