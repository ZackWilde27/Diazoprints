Addons

Addons allow you to extend a language module, this is how the various libraries are added, so programmers have the option to include them when working on a project

One caveat with addons is that they must be written in the .hmodule format, you can't write an addon in JSON or my custom format

Settings

Just like modules, addons have a settings area, but this time around there's only 1 setting needed for addons

struct Settings
{
    addon-decl="#include <my-addon.h>\n"
};

addon-decl

This is code put at the start of the exported file, which actually includes the library in question.
For C libraries, this would be something like '#include <the_library>', for Python libraries this would be something like 'import the_library'

Everything Else

From there, the rest of the file is exactly like a header file module, you can define new types, nodes, classes/structs, whatever that library includes in it
